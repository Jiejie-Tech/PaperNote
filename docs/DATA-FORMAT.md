# PaperNote 数据格式

## 概览

PaperNote 使用 `.papernote` 保存单个笔记本。当前 `FormatVersion` 为 **17**。Windows 与 Android 共用领域模型、页面对象、PDF 学习数据、录音元数据、波形峰值、备份和 PaperInk 墨迹格式。

`.papernote` 是本地文件，不依赖账号或网络服务。导入 PDF 后，PaperNote 会把需要长期保留的页面图像和可用的 PDF 文本/导航信息写入笔记文件；原始 PDF 不会被修改。

## 笔记本结构

每个笔记本包含唯一 ID、标题、时间、文件夹、标签、页面顺序、当前页面、回收站状态、纸张收藏和文档级目录。每个页面可包含：

- 页面标题、尺寸、纸张模板、颜色、标签、书签和手工大纲级别；
- PaperInk 墨迹与兼容 Windows 旧数据的 ISF；
- 文本、图片和形状对象；
- 页面图层、活动图层和对象/笔迹的图层引用；
- PDF 或图片背景、来源、页码、旋转和裁剪；
- PDF 来源 SHA-256 指纹、文本层内容和内部链接；
- 页面文字评论；
- 已保存的 OCR/手写识别文本；
- 本地录音元数据和时间标记。

## PaperInk 与 ISF

PaperInk 记录笔迹 ID、工具、颜色、宽度、透明度、图层 ID，以及位置、压力、倾角和时间等采样信息。Windows 与 Android 都读写 PaperInk。

Windows 仍保留 ISF 兼容字段。旧文件若只有 ISF，建议先用新版 Windows 打开并保存一次以补充 PaperInk；Android 不直接解析仅含 ISF 的旧笔迹。

Windows 画布会把 PaperInk 笔迹 ID、图层 ID 和透明度写入 WPF Stroke 自定义属性；在 PaperInk 与 WPF/ISF 之间往返时保持身份、图层归属和透明度，避免保存后选区身份变化或图层丢失。

加载时会规范化空或重复笔迹 ID、空点、NaN/无穷坐标、异常宽度/透明度/压力/倾角和负时间戳。指向不存在图层的笔迹会转移到活动图层。

## 页面对象、图层与导航

对象使用平台无关坐标，支持文本、图片和多种形状，并保存位置、大小、旋转、线条、填充、透明度、锁定、隐藏、组合、图层和页面链接字段。

加载时会修复空或重复页面、图层、对象、目录、链接和评论 ID；无效对象/笔迹图层引用会迁移到活动图层。指向不存在页面的对象链接、PDF 链接或目录目标会被清理。不同平台字体和绘制器可能产生轻微视觉差异，但模型字段会保留。

页面复制会为复制内容生成新 ID，并重映射复制页之间的链接。页面提取会建立独立笔记本、生成新页面/对象/墨迹 ID、保留所选 PDF 目录并重映射所选页面之间的内部链接；指向未提取页面的链接不会伪造目标。

## PDF 数据

导入后，PDF 页面以 840 × 1188 JPEG 背景内嵌到 `.papernote`，批注仍作为独立墨迹、对象和评论保存。导出时把背景、墨迹和对象合成为普通扁平 PDF，不修改原始 PDF。

格式 16 为每个 PDF 页面保存：

- `BackgroundSourceFingerprint`：原 PDF 完整内容的 SHA-256 指纹；
- 原文件显示名与源页码；
- `PdfText`：原 PDF 已存在的文本层内容；
- `PdfLinks`：内部 GoTo 链接、0–1 归一化矩形、源 PDF 目标页码和可解析的 PaperNote 目标页面 ID；
- 旋转和裁剪值；
- 页面评论与其他独立批注。

文档级 `OutlineEntries` 保存原 PDF 目录/书签的标题、1–6 级层级、源页码和可解析目标页面。内部链接只有目标 PDF 页也导入当前笔记时才能解析为 PaperNote 页面；之后导入同一来源的目标页时可重新解析。

扫描版 PDF 若没有文本层，`PdfText` 为空，但页面图像、批注、来源指纹和导航数据仍可正常保存。当前格式不保存可编辑 PDF 表单，也不会把 PaperNote 评论写回原 PDF。

格式 17 在格式 16 的 PDF 字段基础上新增录音 `WaveformPeaks`。旧文件没有该字段时按空波形处理，新版保存后会升级到格式 17。

## PDF 导入缓存

导入任务缓存不属于 `.papernote` 数据格式，位于平台本地缓存目录。任务 ID 由 PDF 完整 SHA-256 内容指纹、规范化页码集合和渲染尺寸确定；目录包含暂存的 `source.pdf`、逐页 JPEG 与原子写入的 `manifest.json`。

取消或失败会保留已完成页，重新选择同一内容和范围时续接。默认缓存上限为 1 GB，超过 30 天的旧任务会在清理时优先删除。缓存删除不会破坏已经写入 `.papernote` 的页面。

## 本地录音

录音文件保存在资料库的 Audio 目录，笔记中只保存相对本地路径、显示名、MIME 类型、时长、大小、创建时间、时间标记和归一化波形峰值。标记可关联笔迹 ID；`.papernote` 不嵌入音频本体。

Windows 录音为 WAV，保存后由本地 WAV 解析器提取峰值；Android 为 MPEG-4/AAC，在录音期间读取本机麦克风振幅。两端都不会上传音频或调用联网服务。

`WaveformPeaks` 保存 `0` 到 `1` 的浮点峰值，最多 2048 个；超过上限时按相邻峰值取最大值逐步压缩。加载时会移除 NaN/Infinity、裁剪越界值、修复重复/空录音与标记 ID、负时长/大小/偏移并按时间排序；超过录音时长的标记会被约束。

```json
{
  "WaveformPeaks": [0.1, 0.5, 0.9]
}
```

播放位置根据录音时长归一化；经过关联 `StrokeId` 的时间标记后，客户端可在限定时间窗内高亮对应笔迹。波形和高亮都是可重建的本地辅助数据，不改变原始音频。

## 原子保存与崩溃恢复

保存流程先写 `.papernote.tmp`，重新解析验证后再替换正式文件。启动时仅在临时草稿比正式文件更新且可读时自动恢复；正式文件较新时不会被覆盖。

恢复中心只读取资料库目录内的 `.papernote` 或 `.papernote.tmp`。抢救副本会使用新的文档 ID 和文件名，原损坏文件不会被覆盖或删除。过大、空、截断或无法解析的文件只显示安全的只读诊断和最多 4,000 字符原始预览。

## 整库备份格式 3

整库备份扩展名为 `.pnbak`，内部是 ZIP 容器，包含：

- `manifest.json`；
- `notebooks/` 下的笔记；
- `backups/` 下的历史版本；
- `audio/` 下的录音附件。

清单为每个文件记录相对路径、长度和 SHA-256。导入前会检查格式版本、条目数量、总大小、重复路径、路径穿越、清单与 ZIP 是否一致，以及每个笔记/历史版本/音频的长度和哈希。所有内容先在临时目录验证，成功后才写入资料库。

## 兼容与迁移原则

- 读取器忽略不认识的可选字段；新增字段提供安全默认值。
- 破坏性结构调整必须提升 `FormatVersion` 并提供迁移路径。
- 当前加载器会规范化 PDF 指纹、文本、链接坐标、评论时间/颜色、目录层级，以及格式 17 的波形峰值。
- 当前加载器会修复常见畸形数据，但不会静默覆盖原损坏文件。
- 不应在未备份时用旧版客户端批量覆盖新版文件。

## 数据位置

- Windows：由桌面应用资料库设置和工作区状态决定。
- Android：应用私有数据目录；卸载或清除应用数据会删除内容。

跨设备迁移请使用应用内单笔记导出、页面提取或整库备份，不要直接复制 Android 私有目录。


## PNENC2 加密容器

加密不会改变内部 JSON 的 `FormatVersion = 17`。启用密码保护后，整个 JSON 文档被包装在独立的 `PNENC2` 二进制容器中：

`PNENC2 magic | metadataLength | metadata JSON | salt(16) | nonce(12) | tag(16) | ciphertext`

- 算法：AES-GCM，256 位密钥，16 字节认证标签。
- 密钥派生：PBKDF2-SHA256，210,000 次，随机 16 字节 salt。
- nonce：每次保存随机生成 12 字节。
- 密码：至少 8 个字符，不写入任何文件或设置。
- 可见元数据：ID、标题、修改/最近打开时间、文件夹、封面、收藏、置顶、回收站状态、页数。
- 元数据不是机密内容，但作为 AES-GCM AAD 认证；被篡改后正文无法解密。
- 机密内容：页面、笔迹、对象、PDF 文本/图片、批注、录音元数据等全部位于密文中。
- 兼容：加载器继续支持旧 `PNENC1`；下次保存会写为 `PNENC2`。
- 原子性：新内容先写入临时文件并重新解密验证，成功后才替换原文件。
- 恢复：历史版本在替换当前文件前先完成格式与密码验证，失败时保留当前文件原样。
