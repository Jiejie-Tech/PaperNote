# PaperNote 数据格式

## 概览

PaperNote 使用 .papernote 保存单个笔记本。当前 FormatVersion 为 **15**。Windows 与 Android 共用领域模型、页面对象、录音元数据、备份和 PaperInk 墨迹格式。

## 笔记本结构

每个笔记本包含唯一 ID、标题、时间、文件夹、标签、页面顺序、当前页面、回收站状态和纸张收藏。每个页面可包含：

- 页面标题、尺寸、纸张模板、颜色、标签和大纲级别；
- PaperInk 墨迹与兼容 Windows 旧数据的 ISF；
- 文本、图片和形状对象；
- 页面图层、活动图层和对象/笔迹的图层引用；
- PDF 或图片背景、来源、页码、旋转和裁剪；
- 已保存的 OCR/手写识别文本；
- 本地录音元数据和时间标记。

## PaperInk 与 ISF

PaperInk 记录笔迹 ID、工具、颜色、宽度、透明度、图层 ID，以及位置、压力、倾角和时间等采样信息。Windows 与 Android 都读写 PaperInk。

Windows 仍保留 ISF 兼容字段。旧文件若只有 ISF，建议先用新版 Windows 打开并保存一次以补充 PaperInk；Android 不直接解析仅含 ISF 的旧笔迹。

Windows 画布会把 PaperInk 笔迹 ID、图层 ID 和透明度写入 WPF Stroke 自定义属性；在 PaperInk 与 WPF/ISF 之间往返时保持身份、图层归属和透明度，避免保存后选区身份变化或图层丢失。

加载时会规范化空或重复笔迹 ID、空点、NaN/无穷坐标、异常宽度/透明度/压力/倾角和负时间戳。指向不存在图层的笔迹会转移到活动图层。

## 页面对象与图层

对象使用平台无关坐标，支持文本、图片和多种形状，并保存位置、大小、旋转、线条、填充、透明度、锁定、隐藏、组合和图层字段。

加载时会修复空或重复页面、图层和对象 ID；无效对象/笔迹图层引用会迁移到活动图层。不同平台字体和绘制器可能产生轻微视觉差异，但模型字段会保留。

## PDF

导入后，PDF 页面以 840 × 1188 JPEG 背景内嵌到 `.papernote`，批注仍作为独立墨迹和对象保存。导出时把背景、墨迹和对象合成为普通扁平 PDF，不修改原始 PDF。当前格式保存来源名、原页码、旋转和裁剪值，但不保存可编辑 PDF 表单。

导入任务缓存不属于 `.papernote` 数据格式，位于平台本地缓存目录。任务 ID 由 PDF 完整 SHA-256 内容指纹、规范化页码集合和渲染尺寸确定；目录包含暂存的 `source.pdf`、逐页 JPEG 与原子写入的 `manifest.json`。取消或失败会保留已完成页，重新选择同一内容和范围时续接。默认缓存上限为 1 GB，超过 30 天的旧任务会在清理时优先删除。

## 本地录音

录音文件保存在资料库的 Audio 目录，笔记中只保存相对本地路径、显示名、MIME 类型、时长、大小、创建时间和时间标记。标记可关联笔迹 ID。

Windows 录音为 WAV，Android 为 MPEG-4/AAC。加载时会修复重复/空录音与标记 ID、负时长/大小/偏移并按时间排序；超过录音时长的标记会被约束。

## 原子保存与崩溃恢复

保存流程先写 .papernote.tmp，重新解析验证后再替换正式文件。启动时仅在临时草稿比正式文件更新且可读时自动恢复；正式文件较新时不会被覆盖。

恢复中心只读取资料库目录内的 .papernote 或 .papernote.tmp。抢救副本会使用新的文档 ID 和文件名，原损坏文件不会被覆盖或删除。过大、空、截断或无法解析的文件只显示安全的只读诊断和最多 4,000 字符原始预览。

## 整库备份格式 3

整库备份扩展名为 .pnbak，内部是 ZIP 容器，包含：

- manifest.json；
- notebooks/ 下的笔记；
- backups/ 下的历史版本；
- audio/ 下的录音附件。

清单为每个文件记录相对路径、长度和 SHA-256。导入前会检查格式版本、条目数量、总大小、重复路径、路径穿越、清单与 ZIP 是否一致，以及每个笔记/历史版本/音频的长度和哈希。所有内容先在临时目录验证，成功后才写入资料库。

## 兼容与迁移原则

- 读取器忽略不认识的可选字段；新增字段提供安全默认值。
- 破坏性结构调整必须提升 FormatVersion 并提供迁移路径。
- 当前加载器会修复常见畸形数据，但不会静默覆盖原损坏文件。
- 不应在未备份时用旧版客户端批量覆盖新版文件。

## 数据位置

- Windows：由桌面应用资料库设置和工作区状态决定。
- Android：应用私有数据目录；卸载或清除应用数据会删除内容。

跨设备迁移请使用应用内单笔记导出或整库备份，不要直接复制 Android 私有目录。
