# PaperNote 数据格式

> 当前内部版本：`FormatVersion 19`。文件扩展名为 `.papernote`，Windows 与 Android 共用同一模型。

## 1. 文件层级

普通 `.papernote` 是 UTF-8 JSON 文档；启用密码后，JSON 被放入 PNENC2 加密容器。主要层级为：

- `NotebookDocument`：标题、目录、标签、收藏、当前页、纸张预设和页面列表。
- `NotebookPage`：纸张/PDF 背景、PaperInk、页面对象、图层、OCR/PDF 文本、评论、链接和录音。
- `PaperInkDocument`：跨平台笔迹、点、压力、倾角、工具、颜色、宽度、透明度和图层引用。
- `PageObject`：文本、图片、形状、公式、PDF 表单字段、图片签名及对象组/链接元数据。
- `AudioRecording` / `AudioCue`：本地附件路径、时长、波形、质量、裁剪和笔迹时间点。

## 2. 版本历史

| 版本 | 主要新增数据 |
| --- | --- |
| 1–15 | 基础笔记、页面、对象、图层、PDF 背景、页面批量、模板、目录、链接、搜索、备份与恢复等逐步演进。 |
| 16 | PDF 内容指纹、页面文本、文本块、内部链接、文字评论和文档目录。 |
| 17 | 录音 `WaveformPeaks` 归一化波形峰值。 |
| 18 | 批量 OCR 文本块、区域、阅读顺序、置信度、更新时间和 `OcrNeedsReview` 校对状态。 |
| 19 | PDF 字/词坐标、PDF 测量比例；表单字段、图片签名、公式 LaTeX；录音采样率、码率、输入设备和播放裁剪范围。 |

加载旧版本时由 Core 统一迁移、补默认值并修复无效引用；保存后写为格式 19。未知的未来版本不会被静默覆盖。

## 3. PaperInk 与旧 ISF

- `Ink` 是 Windows/Android 的主笔迹格式，所有新功能以 PaperInk 为准。
- `InkData` 保留旧 WPF ISF 数据用于兼容；迁移成功后仍避免破坏原字段。
- 笔迹 ID、图层 ID、对象 ID 和组 ID 在复制、提取或合并时重新生成，引用同步重映射。
- 加载器会限制无效坐标、压力、倾角、宽度、透明度和异常集合大小。

## 4. PDF 数据

每个导入页可保存：

- 来源名称、页码、旋转、裁剪和 SHA-256 内容指纹；
- 原文、按阅读顺序排列的文本块及字/词坐标；
- 原 PDF 目录/书签、内部 GoTo 链接和页面评论；
- 独立 PaperNote 笔迹/对象批注；
- 测量比例和单位；
- 覆盖层表单字段、复选框、必填状态和图片签名。

这些内容不会回写原 PDF。页面范围提取和笔记本合并会生成新 ID，并重映射可解析的内部链接与目录目标；无法解析的外部页面链接会清除。

## 5. OCR 与公式

- 页面保存 `OcrText`、`OcrBlocks`、平均置信度、更新时间和待校对状态。
- 每个识别块包含文本、归一化区域、阅读顺序和置信度。
- 公式作为标准页面对象保存，`FormulaLatex` 使用正常的单反斜杠 LaTeX 命令；SVG 是导出结果，不要求嵌入文档。

## 6. 本地录音

- 音频文件作为本地附件保存，文档只保存相对/受控路径和元数据。
- 波形最多保存 2048 个归一化峰值。
- 格式 19 保存采样率、码率、输入设备说明、`TrimStartMilliseconds` 和 `TrimEndMilliseconds`。
- 裁剪字段只影响播放范围，不改写原音频文件。
- 时间点可关联笔迹 ID；页面复制、提取和笔记本合并时会重映射关联笔迹。

## 7. 原子保存、恢复与容量保护

- 保存到临时文件，重新读取验证成功后再替换正式文件。
- `.tmp`、自动恢复副本和历史版本采用受控目录；恢复中心支持只读检查与另存抢救。
- 空间估算会考虑 JSON、图片、PDF 背景、OCR、笔迹和录音附件，避免明显超出可用空间时继续写入。
- 200 MB/500 页 PDF 是导入边界，不代表每台低内存设备都能在相同速度下处理。

## 8. 整库备份格式 3

备份包包含清单、笔记文件、历史版本和音频附件。每个条目记录长度和 SHA-256；导入前校验路径、数量、长度和摘要，防止路径穿越、截断或静默篡改。

## 9. PNENC2 加密容器

- PNENC2 使用基于密码的密钥派生和认证加密；密码不写入文件或应用设置。
- 加密外壳版本与内部 `FormatVersion` 分开演进：当前组合为 PNENC2 + 格式 19。
- 旧 PNENC1 文件仍可打开并迁移。
- 忘记密码无法找回；图片签名不是加密密钥或数字证书。

## 10. 本地资源包与扩展包

- 素材/模板资源包是 ZIP + JSON 清单，包含数量与 SHA-256；导入时重新生成 ID、跳过重复项并拒绝损坏包。
- 扩展包仅声明受限文字命令，导入时校验命令类型、参数、数量和正则表达式；运行时限制输入输出大小与正则超时。
- 两类包都不应包含签名密钥、私人笔记或任意可执行代码。

## 11. 兼容原则

1. 新字段必须有安全默认值。
2. 迁移不能修改原 PDF、原音频或用户未选择覆盖的文件。
3. 所有跨页面引用在复制、提取、导入和合并时必须验证或重映射。
4. 损坏输入应明确失败，不以空数据覆盖现有笔记。
5. 格式变更必须增加 Core 往返测试并同步本文件。
