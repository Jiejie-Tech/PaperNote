# PaperNote 数据格式

## 概览

PaperNote 使用 `.papernote` 保存单个笔记本。当前 `FormatVersion` 为 **14**。Windows 与 Android 共用领域模型、页面对象、备份和 PaperInk 墨迹格式。

## 主要结构

笔记本包含：

- 唯一标识、标题、创建和修改时间；
- 页面顺序和当前页面；
- 页面名称、尺寸、纸张模板与背景；
- 墨迹数据；
- 文本、图片和形状对象；
- PDF 页面来源及相关渲染信息；
- 向后兼容所需的格式版本信息。

## 墨迹兼容

### PaperInk

PaperInk 是跨平台墨迹表示，记录笔画工具、颜色、宽度、透明度及采样点。采样点可包含位置、压力和倾角。

Android 原生输入会写入 PaperInk。Windows 可读取和编辑这些笔画。

### Windows ISF

Windows 版继续保留 ISF，以兼容已有桌面笔记和 WPF 原生墨迹能力。保存时同时维护 PaperInk，使新数据可在 Android 使用。

若旧文件只有 ISF，Windows 打开并保存后可补充跨平台数据。Android 不直接解析仅含 ISF 的旧笔画，因此迁移旧资料时建议先在新版 Windows 客户端打开并保存一次。

## 页面对象

跨平台对象使用与平台无关的坐标和样式信息。目前包括：

- 文本；
- 图片引用或嵌入数据；
- 直线、箭头、矩形、圆形、三角形、菱形、星形；
- 对象位置、大小、旋转、线条和填充样式。

不同平台的字体和渲染器可能造成轻微视觉差异，但对象属性会保留。

## PDF

导入 PDF 后，页面背景和批注数据分别保存。导出 PDF 时再将背景、墨迹和页面对象合成为扁平页面，不修改原始 PDF。

## 备份

整库备份包含多个 `.papernote` 文件及资料库恢复所需信息。恢复前应保留现有资料库的独立备份，以防误覆盖或导入错误。

## 原子保存与并发

存储层使用临时文件和替换策略降低中断写入风险。移动端保存会在等待写锁前捕获目标笔记引用，避免快速切换笔记时把内容写到错误文件。

## 兼容策略

- 读取器应忽略不认识的可选字段。
- 新增字段优先提供安全默认值。
- 破坏性结构调整必须提升 `FormatVersion` 并提供迁移路径。
- 不应在未备份的情况下用旧版客户端批量覆盖新版文件。

## 数据位置

- Windows：由桌面应用的资料库设置和工作区状态决定。
- Android：应用私有数据目录，普通文件管理器不可直接访问。

跨设备迁移请使用应用内导出或整库备份，不要依赖复制 Android 私有目录。
