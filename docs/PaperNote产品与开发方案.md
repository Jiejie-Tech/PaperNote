# PaperNote：Windows 与 Android 本地墨迹笔记应用产品与开发方案

> 版本：2.0
> 日期：2026-07-21
> 目标平台：Windows 10/11 x64、Android 7.0+（API 24+）
> 目标用户：单人、离线优先、触控笔/数位板优先

## 1. 项目定位

PaperNote 是一款面向 Windows 笔记本电脑、Android 手机和平板的个人电子笔记应用，独立实现本地资料库、纸张模板、分页书写、触控笔工具、页面导航、PDF 批注、内容搜索与专注书写体验。

不包含账号、订阅、在线商城、广告、多人协作和自建云服务。所有笔记默认保存在本地，可备份为独立文件。

## 2. 产品目标

1. 在 Windows 电脑、Android 手机和平板及触控笔/数位板上提供稳定的书写体验。
2. 支持独立设计的本地资料库、文件夹、笔记本、页面和模板结构。
3. 支持钢笔、铅笔、荧光笔、橡皮、套索、形状、图片、文本、元素和激光笔。
4. 支持导入 PDF、在 PDF 上批注，并导出扁平 PDF。
5. 自动保存，异常退出可恢复，并能完整备份笔记库。
6. 无需账号和网络即可使用全部核心功能。

## 3. 功能范围

### 3.1 书架

- 全部文稿、最近打开、收藏、文件夹、PDF、快速笔记、回收站
- 网格/列表视图
- 按编辑时间、创建时间、名称、页数或手动顺序排序
- 创建笔记本、快速笔记、文件夹
- 导入 PDF 和图片
- 重命名、复制、收藏、移动、导出、更换封面、删除

### 3.2 新建笔记本

- 名称、封面、纸张模板、颜色、尺寸、方向和默认滚动方式
- 预设尺寸：经典笔记尺寸、A4、A5、Letter、16:9、4:3、方形和自定义
- 模板：空白、横线、方格、点阵、康奈尔、计划表、课程笔记、五线谱及自定义图片模板

### 3.3 编辑器

- 顶部导航栏：返回、侧栏、标题、保存状态、只读、添加页、书签、搜索、导出、更多
- 左侧栏：页面缩略图、书签、大纲
- 页面操作：添加、复制、删除、旋转、改模板、调整顺序、加入大纲、导出当前页
- 浏览模式：纵向连续、横向连续、单页、双页
- 缩放范围：20%～800%，支持适合宽度和适合整页

### 3.4 工具栏

- 撤销、重做
- 铅笔
- 圆珠笔、钢笔、画笔
- 荧光笔
- 精确擦除、整笔擦除、仅擦荧光笔
- 直线、箭头、矩形、圆角矩形、椭圆、三角形和多边形
- 自由套索和矩形套索
- 元素/贴纸库
- 图片、文本框
- 激光点和渐隐轨迹

工具栏规则：单击切换工具，再次单击打开设置；每种笔保存三个粗细快捷档和三个常用颜色。

### 3.5 输入逻辑

- 笔尖书写，笔尾擦除，侧键可映射套索或橡皮
- 手指默认只负责平移和缩放
- 鼠标左键使用当前工具，中键或空格+左键平移
- 检测到触控笔后抑制手掌触摸；笔抬起后延迟约 150ms 恢复触摸
- 可配置手指书写、鼠标书写、压感曲线和平滑度

### 3.6 PDF

- PDF 作为新笔记本导入，或插入现有笔记本指定位置
- 支持选择部分页面、旋转和保留原文件
- PDF 页面作为背景层，笔迹和对象位于其上方
- PDF 源文件与逐页 JPEG 使用内容指纹缓存；异步导入显示进度，可取消并在失败后续接
- 导出扁平 PDF、PNG 和可继续编辑的 PaperNote 文件

### 3.7 搜索和识别

- 搜索笔记本名、文件夹名、键入文本、PDF 文本、手写识别文本、大纲和书签
- 页面停止编辑约 2 秒后进入后台识别队列
- 套索选中笔迹后可转换为文本或复制识别结果
- 使用 Windows 本地手写识别能力，不依赖在线 AI

### 3.8 数据安全

- 操作停止约 500ms 后自动保存
- 页面切换、失焦和退出时立即保存
- 临时文件写入成功并验证后再替换正式文件
- 上次异常退出时自动检测并恢复
- 每日完整备份，默认保留 14 天
- 备份目录可设为移动硬盘或用户现有同步文件夹

## 5. 技术选型

### 5.1 客户端技术

- C#、.NET 10、WPF、.NET MAUI、XAML
- PaperNote.Core 共享领域模型、存储、备份、搜索、PDF 与 PaperInk 墨迹
- Windows 使用 WPF InkCanvas 与 ISF，并同步维护 PaperInk
- Android 使用原生 MotionEvent 采样与 MAUI 响应式界面
### 5.2 数据与文档

- Microsoft.Data.Sqlite / SQLite：书架、页面和搜索元数据
- ISF：每页可编辑笔迹
- JSON：页面对象、工具属性和清单
- 独立资源文件：图片、PDF、音频和缩略图
- ZIP：导出的 `.papernote` 可传输笔记本包

### 5.3 PDF 与识别

- PDFtoImage 5.2.1（PDFium + SkiaSharp）：当前桌面版 PDF 页面读取与渲染
- PDFsharp 稳定版：PDF 输出
- Windows Ink Recognizer：本地手写识别
- SQLite FTS5：全文搜索

## 7. 页面与墨迹架构

每一页从下到上分为：

1. 页面背景
2. PDF 或纸张模板
3. 荧光笔层
4. 普通墨迹层
5. 文本、图片、形状和元素层
6. 选择框、控制点和临时激光笔层

每条笔迹保存工具类型、颜色、基础宽度、透明度、边界和带压力/时间/倾斜信息的点集合。

页面采用虚拟化：当前页完整加载，前后各预加载两页，其他页只保留缩略图，远离视口后释放高分辨率对象。

## 9. 核心数据表

- `Folders`：层级文件夹、名称和排序
- `Notebooks`：标题、封面、页数、收藏和时间
- `Pages`：顺序、尺寸、模板、笔迹路径、对象路径、PDF 来源页和版本
- `Assets`：图片、PDF、音频的路径和校验值
- `SearchEntries`：识别文本、输入文本和 PDF 文本
- `Backups`：备份路径、日期和大小

## 10. 性能目标

- 冷启动目标 2 秒以内
- 普通笔记本打开目标 1 秒以内
- 书写时保持视觉跟手，目标 60FPS
- 支持约 500 页笔记本；单个 PDF 上限 200 MB，一次最多导入 500 页
- 单页约 10000 条笔迹仍能编辑
- 自动保存不得阻塞输入线程
- 150% 和 200% 系统缩放下笔迹位置正确
- 强制结束后尽量只损失最后约 1 秒尚未持久化的操作

## 13. 当前实施决定

- 使用 .NET 10；Windows 保留 WPF 与 InkCanvas，Android 使用 .NET MAUI 和原生触控输入，跨平台逻辑统一放入 PaperNote.Core。
- 正式笔记本采用单个 `.papernote` JSON 容器，每页保存 Base64 ISF 笔迹；当前规模下无需引入数据库。
- 文件保存统一采用“写入临时文件后替换正式文件”，并让损坏的单个笔记本或单页笔迹与其他数据隔离。
- 保持完全离线，不建设账号、订阅、协作、云同步或服务器。
- 阶段三先实现学习场景中最常用的纸张模板、页面复制、图片、文本和形状，再进入 PDF 批注。
- 若未来单本笔记达到数百页或包含大量图片，再将容器升级为 ZIP 包和资源分片，不在当前阶段提前复杂化。
## 14. 行业功能调研基线（截至 2026-07-18）

本项目围绕“纸张感书写 + 页面化笔记本 + 可编辑墨迹 + PDF 批注”的通用数字笔记工作流独立设计，同时针对个人 Windows 笔记本电脑使用场景，去掉账号、订阅、协作和云服务依赖。

截至 2026 年 7 月，同类数字笔记产品普遍覆盖笔记本、白板、模板、内容搜索、文字提取、PDF/图片导入、套索和音频等能力。PaperNote 仅把这些通用用户需求作为能力调研输入，并以独立代码、独立交互和独立视觉实现确定产品优先级：

1. **第一优先级：书写体验。** 低延迟、压感、钢笔、荧光笔、橡皮、套索、缩放和撤销必须先稳定。
2. **第二优先级：纸张与页面。** 笔记本、封面、模板、多页面、缩略图、排序和收藏构成核心组织方式。
3. **第三优先级：资料批注。** PDF 导入、逐页批注、搜索和 PDF 导出是学习场景的关键闭环。
4. **第四优先级：数字化编辑。** 手写转文字、形状整理、Smart Ink 类重排、图片和文本对象在核心体验稳定后加入。
5. **暂不实施：在线协作、账号、订阅、云同步、AI 和多人共享。** 私人单机版不为这些功能引入服务器复杂度。

技术上继续采用 Windows 原生墨迹路线。WPF `InkCanvas` 可直接承载鼠标和触控笔笔迹；微软文档也明确说明墨迹系统能够采集压力、笔尖形状、颜色、橡皮和选择等信息。后续如默认渲染无法满足铅笔、钢笔纹理和更低延迟要求，再升级到自定义 `DynamicRenderer`，无需推翻现有文档和页面层。

调研来源：

- [Microsoft：WPF Ink Input Control](https://learn.microsoft.com/en-us/dotnet/desktop/wpf/advanced/creating-an-ink-input-control)
- [Microsoft：Custom Rendering Ink](https://learn.microsoft.com/en-us/dotnet/desktop/wpf/advanced/custom-rendering-ink)

## 31. 2026-07-20 关闭流程与稳定性修复

### 31.1 无法关闭的根因

旧关闭事件在 WPF 界面线程中调用异步保存后，再通过同步等待取得结果。文件写入完成后的异步延续需要回到同一个界面线程，而界面线程又正在等待该任务，形成死锁，因此窗口可能一直无法关闭。

### 31.2 新关闭状态机

新的关闭流程分为两次关闭事件：

1. 第一次收到关闭请求时取消本次关闭，并设置“正在关闭”状态；
2. 停止自动保存计时器，并暂时禁止书架和编辑器继续接收操作；
3. 通过异步方式等待未保存笔记落盘；
4. 尝试保存标签和书架排序等工作区状态；工作区设置失败只记录状态，不阻止笔记保存和退出；
5. 保存完成后允许第二次关闭事件直接通过；
6. 关闭完成后停止计时器、解除笔迹事件，避免后台回调继续引用窗口。

若笔记保存失败，用户仍可选择“仍然退出”或返回应用继续处理；返回后会恢复界面操作和自动保存重试。连续点击关闭不会重复启动保存流程。

### 31.3 同时修复的风险点

- 自动保存、手动保存和关闭前保存统一串行执行，避免同一本笔记被并发写入；
- 快速切换标签时只允许一个打开流程运行，避免后完成的旧请求覆盖当前界面；
- 关闭当前标签的多个入口统一捕获保存异常，避免 async void 异常导致程序崩溃；
- 启动异步读取的各阶段检查关闭状态，窗口关闭后不继续创建默认笔记、恢复状态或刷新界面；
- 后台工作区保存改为安全的 fire-and-forget 包装，不产生未观察异常。

### 31.4 后台验收

隐藏 WPF 测试使用系统临时目录构造独立笔记本和工作区文件，并创建透明、移出屏幕且不显示在任务栏的测试窗口。测试会：

- 写入未保存标题后调用真实 Window.Close；
- 连续调用两次 Close 验证防重入；
- 推动 WPF Dispatcher，确认 10 秒内完成关闭而不死锁；
- 重新读取临时 .papernote，确认退出前修改已保存；
- 检查工作区状态文件已写入；
- 检查关闭后自动保存计时器已停止；
- 清理全部临时测试文件，不读取或改写正式笔记。

Debug 与 Release 的存储冒烟测试、隐藏 WPF 测试和编译均通过，且正式笔记及旧快速笔记 SHA-256 保持不变。

- Windows 与 Android 均允许对单本笔记启用、更改和关闭密码保护。
- 密码至少 8 个字符，只保留在当前进程内存，不写入文件、设置或日志，也不提供找回机制。
- 书架需要在未解锁时显示基本管理信息，因此标题、文件夹、封面、页数、收藏/置顶和回收站状态作为可见元数据；正文与附件数据全部加密。

### 37.2 技术实现

- 内部文档继续使用 `FormatVersion = 17`，外层使用 `PNENC2`。
- AES-GCM 256 位密钥；PBKDF2-SHA256 210,000 次；16 字节 salt、12 字节 nonce、16 字节 tag。
- 元数据作为 AAD 参与认证；任何元数据或密文篡改都会导致解锁失败。
- 保存采用临时文件写入、重新解密/解析验证和原子替换；普通保存接口拒绝覆盖加密文件。
- 历史恢复先验证备份，再替换当前文件；支持启用保护前的明文版本、更改密码前的旧密码版本和关闭保护后的加密版本。
- 整库备份按字节保留 PNENC1/PNENC2，不解密私人内容。
