# PaperNote 离线本地竞品差距分析

> 评估日期：2026-07-22
> 评估对象：PaperNote 1.0.0（Windows、Android）
> 对比范围：只比较不依赖账号、云同步、多人协作、在线素材商店和在线 AI 的本地使用能力。

## 1. 结论先行

PaperNote 已经越过了“演示项目”阶段：它拥有真实的本地资料库、分页笔记、手写墨迹、页面对象、PDF 导入批注、自动保存、回收站、历史版本、整库备份，以及 Windows/Android 共用的数据模型。它最有价值的差异化不是功能数量，而是：**无账号、无广告、数据默认留在设备、本地文件可迁移、开源且可自行维护。**

但如果目标是成为可长期替代 Goodnotes、Notability 的离线主力笔记工具，目前仍有五个决定性短板：

1. **高级选区主干已补齐，交互精度仍需深化**：Windows 与 Android 已具备自由形状混合套索、八类筛选、混合变换、批量样式和跨页复制/移动；仍缺少选区导出、键盘微调、更精细的控制柄与触控反馈。
2. **书写质感与工具深度仍有提升空间**：局部/整笔橡皮、平滑、透明度、压感曲线和常用笔预设已经具备；标尺、自动形状识别、更多设备适配和长期书写体验仍需继续打磨。
3. **本地检索基础已完成，但识别引擎尚未内置**：结构化全文搜索可索引标题、标签、文字对象、资料来源以及已保存的 OCR/手写识别文本；扫描 OCR、手写转文字和数学识别仍需要接入许可合适的离线引擎。
4. **PDF 仍是基础批注级**：缺少成熟的文本搜索、批注列表、页面重排/提取/合并、链接和大纲导航，以及 Android 端更完整的旋转、裁剪和批量页操作。
5. **大型 PDF 的基础导入链路已补齐，极限规模和长期设备验证仍不足**：Windows/Android 已有 200 MB/500 页边界、逐页进度、取消、缓存与续接，但低存储、长时间运行和多品牌真机仍需持续压测。

综合判断：

- **Windows 端**：已经是可用的轻量本地手写/PDF 笔记工具，成熟度约为 `3/5`。
- **Android 端**：核心书写、常用对象编辑、图层、录音和恢复链路已经可用，成熟度约为 `3/5`。
- **跨平台数据基础**：PaperInk 与共享 Core 是明显优势，约为 `4/5`。
- **跨平台功能一致性**：常用编辑与数据模型已基本统一，高级工具仍有差异，约为 `3/5`。
- **本地数据控制**：是 PaperNote 当前最强项，约为 `5/5`。

这里的分数代表工作流成熟度，不是代码质量评分，也不是商业产品绝对排名。

## 5. 分类差距分析

### 5.1 手写基础：已经可用，但还不够“像纸”

#### PaperNote 已有

- 钢笔和荧光笔
- 粗细和颜色调整
- 撤销与重做
- 触控笔压力输入
- Android 倾角采样和橡皮端识别
- 页面平移和双指缩放
- Android 手指书写/单指平移切换
- PaperInk 跨平台墨迹
- Windows ISF 兼容保留

#### 与 Goodnotes、Notability 的差距

1. **笔型不够丰富**：缺少铅笔、圆珠笔/中性笔差异、毛笔或书法笔、虚线笔等。
2. **笔迹参数仍不够专业**：透明度、压感曲线和平滑已经具备，但缺少笔尖形状、速度响应、起收笔和最小/最大宽度等细化。
3. **书写稳定性缺少可感知指标**：需要持续测量首笔延迟、移动延迟、抖动、断线、丢点、缩放后笔迹一致性。
4. **局部修整已有基础但交互仍可提升**：局部橡皮可切分笔迹并保留图层，但缺少按工具类型过滤和更清晰的擦除范围反馈。
5. **快捷笔工作流仍可深化**：常用笔预设已经具备，但预设管理、自定义工具栏和跨设备一致性仍可增强。
6. **触控笔快捷键不完整**：侧键、长按、橡皮端和工具切换应支持用户自定义。
7. **防误触仍需设备级验证**：尤其是不同 Android 品牌、主动笔协议、掌压和多指手势冲突。

#### 建议

优先做“书写引擎 2.0”：局部橡皮、笔迹平滑、常用笔预设、透明度、压感曲线、设备兼容性测试和性能指标。不要先堆十几种装饰性笔刷。

### 5.2 套索和对象编辑：常用闭环已统一，高级选择仍有差距

#### Windows 已有

- 套索选择
- 对象复制
- 移动、缩放、旋转
- 组合、锁定
- 对齐和分布
- 层级调整
- 文本、图片和形状编辑

#### Android 当前状态

- 已支持自由形状混合套索，可选择笔迹、文字、图片和形状，并提供八类筛选、统一移动/缩放/旋转、复制、删除、组合/取消组合、层级、锁定、批量样式和跨页复制/移动。
- 文本、图片和形状的跨平台字段可以无损保存，图层显隐和透明度也会参与渲染。
- 主要差距已收敛为选区导出、键盘微调、更精细的控制柄命中、触控反馈和更复杂的组合编辑。

#### 建议

把对象编辑能力下沉为跨平台的 Core 命令和几何算法，由 Windows 与 Android 分别负责交互和渲染。至少统一：

- 点选和套索
- 选择框与控制柄
- 移动、缩放、旋转
- 复制、删除
- 颜色、线宽和透明度修改
- 前移、后移、置顶、置底
- 组合、取消组合、锁定
- 撤销/重做命令模型

基础闭环已经完成；后续重点应放在自由套索、笔迹级批量编辑和触控细节，而不是重复实现已有对象命令。

### 5.3 橡皮、选择和手写编辑：缺少精细控制

Goodnotes、Notability 等产品的优势不只是“有套索”，而是选中后能快速完成复制、转换、缩放、颜色修改、截图、移动到其他页面等动作。

PaperNote 还缺少：

- 擦除荧光笔但保留钢笔，或按对象类型过滤
- 套索只选择手写、只选择文本、只选择图片等过滤器
- 选中笔迹后统一改颜色、粗细和透明度
- 跨页面复制/移动
- 选区保存为图片或常用元素
- 涂抹删除等自然手势
- 误触时的选区保护和点击空白取消逻辑优化

### 5.4 页面、模板和资料库：基础完整，规模化组织不足

#### 已有

- 新建、复制、删除、重命名页面
- 缩略图、页码、快速跳转
- 空白、横线、方格、点阵模板
- 新建、打开、重命名和删除笔记本
- 最近笔记、标题搜索、回收站
- 页面书签和大纲相关数据字段

#### 仍缺

- 用户导入图片/PDF 作为自定义模板
- 模板尺寸、方向、边距、背景色和线距编辑
- 模板收藏、最近使用和分类管理
- 页面批量选择、复制、移动、旋转、导出、删除
- 多页拖拽排序，尤其 Android
- 书签和大纲的完整导航界面
- 文件夹嵌套、标签、收藏、置顶和智能筛选
- 最近打开位置、页面浏览历史和“返回上一位置”
- 多笔记标签页、分屏或同时查看两个页面
- 重复笔记、合并笔记和拆分页范围

如果目标用户是一名学生，最实用的结构是：`课程/文件夹 → 笔记本 → 页面 → 书签/大纲/标签`。不必复制 OneNote 的全部层级，但需要支持学期和课程规模的数据量。

### 5.5 PDF：已有核心闭环，但距离专业批注仍有明显差距

#### 已有

- 导入 PDF 为页面背景
- 在 PDF 页面上书写和添加对象
- 导出包含批注的扁平 PDF
- Windows 已有部分页面旋转、裁剪和重置能力
- Android 可以导入和导出 PDF
- Windows 与 Android 均支持页码范围、200 MB/500 页边界、异步逐页进度和取消
- 以完整内容指纹隔离源文件与页面缓存，取消或失败后可续接；默认 1 GB 上限并清理旧任务

#### 关键不足

1. **PDF 文本层能力不足**：缺少文本搜索、选择、复制和高亮原文。
2. **导航不足**：缺少成熟的目录、大纲、书签、内部链接和最近跳转历史。
3. **页面处理不足**：缺少重排、提取、合并、插入、替换、按范围导出和批量旋转。
4. **批注管理不足**：缺少批注列表、按作者/类型/颜色筛选和点击跳转。
5. **Android 能力不统一**：桌面已有的旋转、裁剪和批量页操作未完整下放。
6. **大文件验证仍不足**：导入进度、取消、逐页缓存、淘汰和失败恢复已经完成，但 200 MB/500 页边界仍缺少低内存真机、低存储和长时间反复导入验证。
7. **导出选择不足**：目前以扁平 PDF 为主，缺少分辨率、压缩、页范围、背景是否包含等选项。
8. **专业能力缺失**：签名、表单填写、测量、标尺、校准比例、批注盖章等尚未覆盖。

建议先把“学习资料 PDF”做深，而不是立即追赶工程审图软件的所有能力。可靠的大文件导入基础已经完成，下一阶段应集中在文本搜索、大纲/书签、页面批量处理和批注列表。

### 5.6 搜索、OCR 和手写识别：当前最大效率差距

PaperNote 目前主要是笔记标题搜索。对于几十本甚至几百本课堂笔记，这不足以支持长期使用。

#### 建议的本地检索分层

1. **第一级：文本索引**
   - 索引笔记本标题、页面标题、文本对象和 PDF 文本层。
   - 支持结果片段、页码、命中高亮和点击跳转。
2. **第二级：本地 OCR**
   - 对扫描 PDF 和图片进行离线 OCR。
   - OCR 结果保存为可重建索引，不破坏原始页面。
3. **第三级：手写搜索**
   - 对中文、英文、数字和常见符号进行本地识别。
   - 保留识别置信度，避免把错误结果写回原笔迹。
4. **第四级：手写转文字与结构化编辑**
   - 选中一段笔迹后转为文本。
   - 后续再扩展数学公式和图形识别。

#### 实施原则

- 不应自研完整识别模型；应评估可离线部署、许可可接受的 OCR/手写识别引擎。
- 索引必须可删除、可重建，并明确存放位置。
- 搜索功能不能依赖联网，也不能默认上传用户笔记。
- 识别失败不能影响原始笔迹和文件可打开性。

### 5.7 录音和课堂场景：不是基础刚需，但对学生价值很高

Notability 的代表优势是录音与笔迹时间轴：回放录音时可以定位当时写下的内容。Goodnotes 也已具备录音类工作流。

PaperNote 已完成本地录音基础版：Windows 与 Android 都支持页面级录音、暂停/继续、播放、重命名、删除、命名时间标记和书写时自动记录时间点，音频附件也进入备份格式 3。

仍需补齐：

- 录音波形或更清晰的时间条
- 点击笔迹跳到对应录音时间
- 播放时突出显示当时的笔迹
- 外接麦克风选择和音量提示
- 存储空间预估、压缩质量选择和更强的异常中断恢复

不必做云端转写，也不必联网。

### 5.8 导入、导出、备份和数据安全：核心自救链路已完成

#### PaperNote 的优势

- 单笔记导入导出
- 整库备份和恢复
- 本地自动保存
- 回收站
- 历史版本
- 跨平台 PaperInk
- Android 无 INTERNET 权限

#### 已补强

- 临时文件写入、重新解析验证和原子替换
- 启动时自动恢复较新的可读草稿
- Windows/Android 恢复中心、损坏检测、只读诊断和另存抢救副本
- 备份格式 3 的清单、长度、SHA-256、重复路径和路径穿越校验
- Android 切后台或页面销毁前保存笔记和录音
- 畸形 ID、图层引用、非有限墨迹值和录音时间的加载修复

#### 仍需补强

- 自动备份保留策略可配置
- 更系统的磁盘空间不足提示与预留空间策略
- 数据格式降级提示和更大规模的跨版本模糊测试
- 卸载前备份提醒进一步强化
- 可选本地加密或密码保护界面

一个本地优先产品最重要的承诺不是“数据在本地”，而是“数据在本地并且不容易丢”。

### 5.9 Windows 与 Android 一致性：共享格式是优势，交互能力尚未统一

PaperNote 已经采用共享 Core 和 PaperInk，这是后续扩展的正确基础。但目前存在“格式一致、能力不一致”的问题。

需要建立跨平台能力合同：

- 同一对象在两个平台都能创建、查看、编辑和删除
- 同一笔迹在两个平台渲染尺寸、颜色和压感结果一致
- 同一 PDF 页裁剪、旋转和缩放结果一致
- 撤销/重做语义一致
- 旧版本文件升级后两端结果一致
- 不支持的对象也必须无损保留，不能打开后保存就丢失

建议为每个文件格式版本建立 Windows ↔ Android 往返测试：创建、保存、互相打开、编辑、再次保存，再比较文档模型和渲染快照。

### 5.10 性能和稳定性：墨迹路径已优化，极限规模仍需验证

需要重点测试：

- 单页 1 万、5 万、10 万采样点
- 100、500、1000 页笔记本
- 50 MB、200 MB、500 MB PDF
- 连续书写 30 分钟和 2 小时
- 快速缩放、翻页、撤销重做和批量复制
- 低内存 Android 设备
- 应用切后台、锁屏、来电和进程被回收
- 磁盘空间耗尽和写入权限变化
- 中途拔掉外接存储
- 文件被其他程序占用或用户移动

已经完成：

- 大墨迹空间索引
- Android 视口内按需渲染和预加载边界
- 局部橡皮先筛选附近候选再做线段级命中
- 10,000+ 笔迹索引测试和 60 页/4,800 笔迹/600 对象重复保存压力测试

仍需继续：
- 页面缩略图异步生成和缓存
- PDF 页面分级缓存
- 增量保存和防抖保存
- 大对象图片压缩与分辨率控制
- 撤销栈内存上限
- 后台任务取消和进度报告

### 5.11 可访问性和键盘效率：目前明显弱于成熟桌面应用

建议补齐：

- 所有按钮的可读名称和工具提示
- 完整键盘焦点顺序
- 键盘创建页、翻页、搜索、缩放、切笔、撤销重做
- 可自定义快捷键
- 高对比度和色盲友好配色
- 200%/300% 界面缩放
- 屏幕阅读器读取资料库、页面标题和文本对象
- 触控目标最小尺寸
- Android 横屏、分屏和系统字体放大
- “仅图标”按钮的文字说明模式

## 6. PaperNote 当前真正的优势

与竞品比较时，不应只列缺点。PaperNote 已经有几项值得继续强化的产品定位：

1. **本地优先且无强制账号**：打开即可用，用户自己决定文件位置和备份方式。
2. **开源与可审计**：适合关心隐私、希望长期维护或自行定制的用户。
3. **Windows + Android 共用数据模型**：跨平台不是重新做两个互不兼容的应用。
4. **PaperInk 跨平台墨迹**：已经为后续平台扩展和数据迁移打下基础。
5. **PDF、页面对象和资料库已经形成雏形**：不是单一白板或单页画板。
6. **回收站、历史版本、整库备份已具备基础**：比许多小型开源笔记项目更接近日常使用。
7. **Android 不申请 INTERNET 权限**：这是可以被用户直接验证的隐私承诺。
8. **没有订阅、广告和数据商业化压力**：可以把工程资源放在可靠性和离线体验上。

建议公开定位为：

> 面向 Windows 与 Android 的开源、本地优先手写笔记与 PDF 批注工具。

不要宣传为“完整替代 Goodnotes”，至少在 Android 对象编辑、全文检索、大文档可靠性和 PDF 导航完成之前不宜这样承诺。

5. 从历史版本恢复。

## 10. 最终判断

PaperNote 现在最接近的是：

> 一个已经具备真实本地资料库、手写、页面对象和 PDF 批注能力的 Windows/Android 开源笔记应用。

它还不是 Goodnotes 的完整离线替代品，主要不是因为缺少云端能力，而是因为：

- Windows 与 Android 已完成自由形状混合套索、八类筛选、批量样式和跨页操作；选区导出、键盘微调与触控反馈仍可增强；
- 笔迹编辑和书写质感仍不够成熟；
- 本地全文检索和识别结果索引已经可用，但尚未内置扫描 OCR、手写转文字和数学识别引擎；
- PDF 文档级导航和页面处理不足；
- 墨迹大页面、数据自救和大型 PDF 可恢复导入已有基础保证，但极限页数、低存储和长期真机验证仍不足。

最合理的开发顺序不是继续横向增加零散按钮，而是：

1. 继续细化 Android 与 Windows 的高级编辑一致性；
2. 把书写和橡皮做到稳定、顺手；
3. 在已完成的保存恢复、墨迹索引和 PDF 缓存基础上，继续验证极限规模；
4. 深化 PDF 学习资料工作流；
5. 在现有本地搜索和录音基础上接入可离线部署的 OCR、手写识别与音频-笔迹同步回放。

当前 P0 中的对象编辑、书写主体、恢复安全、墨迹性能和大型 PDF 可恢复导入主体已完成；尚未完成的关键 P0 是更完整的跨平台渲染、低内存真机与极限规模验证。P1/P2 的主要缺口仍是离线识别、PDF 文本层与批注管理、高级几何工具、无障碍和音频同步回放。

## Implementation status — 2026-07-22

本节是当前离线版本能力边界的统一说明。

### 已实现并纳入仓库测试

- Windows 与 Android 共用 PaperInk；支持压力、倾角、平滑、整笔/局部擦除、透明度和图层归属。
- Windows 与 Android 均支持自由形状混合套索，可按全部内容、笔迹、钢笔、荧光笔、对象、文字、图片或形状筛选，并统一移动、缩放、旋转、复制、删除、跨页复制/移动及批量修改颜色、粗细、透明度和笔型。
- 页面图层支持新增、激活、显隐、锁定、透明度、重命名、合并和删除时迁移内容；隐藏内容不会从文件中丢失。
- 文本、图片和形状对象的旋转、透明度、锁定、隐藏、组合和图层字段可跨平台保存。
- 离线搜索覆盖笔记/页面标题、标签、文本对象、已存 OCR 文本、已存手写识别文本和来源名称。
- 保存采用“临时文件写入并验证后替换”；启动时可恢复较新的临时草稿。Windows 与 Android 都提供恢复中心，可只读检查损坏文件并另存为抢救副本，原文件保持不变。
- 读取时会修复空或重复 ID、无效图层引用、非有限墨迹数值及异常录音时间数据。
- 大墨迹页面使用空间索引；Android 按可见视口绘制并用局部候选执行橡皮命中，避免每帧扫描全部笔迹。
- Windows 与 Android 的 PDF 导入支持页码范围、最多 500 页和 200 MB 文件上限；导入异步运行，显示逐页进度并可取消。
- PDF 任务按完整 SHA-256 内容指纹、页码集合和渲染尺寸隔离缓存；取消或失败后再次选择同一文件与范围会续接已完成页面。缓存默认上限 1 GB，并清理超过 30 天的旧任务。
- 导入页面使用 840 × 1188 JPEG 背景并保持原页比例；笔记文件继续内嵌已导入的页面图像，原 PDF 不会被修改。
- 整库备份格式 3 包含笔记、历史版本和音频附件，记录长度与 SHA-256，并在导入前检查重复路径、越界路径、大小和内容完整性。
- Windows 与 Android 均支持页面级本地录音、播放/暂停、重命名、删除、命名时间标记和书写时自动标记。Windows 使用 WAV，Android 使用 MPEG-4/AAC。
- 后台回归覆盖 PDF 真实渲染、缓存全命中、进度完成、取消续接和临时文件清理，以及 10,000+ 笔迹空间查询和 60 页/4,800 笔迹/600 对象重复保存往返。

### 当前明确边界

- OCR 和手写识别结果可以保存并搜索，但仓库尚未内置真正的离线 OCR、手写转文字或数学识别引擎。
- 已提供自由形状混合套索、八类筛选和大批量笔迹样式修改；尚未提供几何吸附、自动形状识别和标尺。
- 录音暂不含波形视图、播放时笔迹高亮、麦克风设备选择和压缩质量控制。
- PDF 尚不含文本层搜索/选择、原文高亮、成熟大纲与内部链接、批注列表、可编辑表单、签名和测量；200 MB/500 页边界仍需更多低内存 Android 真机长期压力验证。
- 完整屏幕阅读器语义、高对比度专项适配、加密设置界面和本地插件机制仍待完善。
- 账号、云同步、联网 AI、遥测、广告和多人协作不在离线版本范围内。
- APK、ZIP、签名密钥、构建输出和私人笔记属于发布产物或本地数据，不提交到源码仓库。
