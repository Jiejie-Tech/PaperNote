# PaperNote 离线本地竞品差距分析

> 评估日期：2026-07-23
> 评估对象：PaperNote 1.0.0（Windows、Android）
> 对比范围：只比较不依赖账号、云同步、多人协作、在线素材商店和在线 AI 的本地使用能力。

## 1. 结论先行

PaperNote 已经越过“演示项目”阶段：它拥有真实的本地资料库、分页笔记、手写墨迹、页面对象、几何规整、混合选区 PNG、可恢复的大型 PDF 导入、PDF 文本/目录/内部链接、批注列表、页面批量处理、本地录音波形与笔迹同步回放、自动保存、回收站、历史版本、整库备份，以及 Windows/Android 共用的数据模型。它最有价值的差异化仍是：**无账号、无广告、数据默认留在设备、本地文件可迁移、开源且可自行维护。**

本轮完成后，原报告中最影响 PDF 学习的 P0 缺口已经明显收敛：

1. **PDF 学习主链路已完成**：带文本层 PDF 可离线搜索；原目录/书签和内部链接可导航；批注可按类型/颜色筛选并添加文字评论。
2. **页面批量主链路已完成**：Windows 与 Android 都可复制、删除、移至首尾、旋转、书签、范围导出和提取为新笔记本。
3. **跨平台数据闭环已完成**：PDF 文本、目录、链接、评论和格式 17 录音波形可随 `.papernote` 在 Windows/Android 之间往返。
4. **识别仍是最大效率短板**：扫描版 PDF/图片离线 OCR、手写转文字和数学识别尚未内置。
5. **竞品级精细体验仍有差距**：基础几何规整、选区 PNG、个人素材库和音频笔迹双向跳转已补齐，但完整标尺、素材分类/导入导出、双页/分屏、完整无障碍和高级 PDF 表单/签名/测量仍待完成。
6. **极限规模验证仍不足**：200 MB/500 页导入边界、缓存与续接已经具备，但低存储、长时间运行和多品牌真机仍需持续压测。

综合判断：

- **Windows 端**：已是可长期使用的轻量本地手写/PDF 学习工具，成熟度约为 `3.8/5`。
- **Android 端**：核心书写、PDF 学习、页面批量、对象编辑、图层、录音和恢复链路已可用，成熟度约为 `3.8/5`。
- **跨平台数据基础**：PaperInk、格式 17 与共享 Core 是明显优势，约为 `4.5/5`。
- **跨平台功能一致性**：常用编辑、PDF 学习与页面批量已基本统一，约为 `4.2/5`。
- **本地数据控制**：仍是 PaperNote 当前最强项，约为 `5/5`。

这里的分数代表工作流成熟度，不是代码质量评分，也不是商业产品绝对排名。

## 5. 分类差距分析

### 5.1 手写基础：已经可用，但还不够“像纸”

#### PaperNote 已有

- 钢笔和荧光笔
- 粗细和颜色调整
- 撤销与重做
- 触控笔压力输入
- Android 倾角采样和橡皮端识别
- 页面平移和双指缩放
- Android 手指书写/单指平移切换
- PaperInk 跨平台墨迹
- Windows ISF 兼容保留

#### 与 Goodnotes、Notability 的差距

1. **笔型不够丰富**：缺少铅笔、圆珠笔/中性笔差异、毛笔或书法笔、虚线笔等。
2. **笔迹参数仍不够专业**：透明度、压感曲线和平滑已经具备，但缺少笔尖形状、速度响应、起收笔和最小/最大宽度等细化。
3. **书写稳定性缺少可感知指标**：需要持续测量首笔延迟、移动延迟、抖动、断线、丢点、缩放后笔迹一致性。
4. **局部修整已有基础但交互仍可提升**：局部橡皮可切分笔迹并保留图层，但缺少按工具类型过滤和更清晰的擦除范围反馈。
5. **快捷笔工作流仍可深化**：常用笔预设已经具备，但预设管理、自定义工具栏和跨设备一致性仍可增强。
6. **触控笔快捷键不完整**：侧键、长按、橡皮端和工具切换应支持用户自定义。
7. **防误触仍需设备级验证**：尤其是不同 Android 品牌、主动笔协议、掌压和多指手势冲突。

#### 建议

优先做“书写引擎 2.0”：局部橡皮、笔迹平滑、常用笔预设、透明度、压感曲线、设备兼容性测试和性能指标。不要先堆十几种装饰性笔刷。

### 5.2 套索和对象编辑：常用闭环已统一，高级选择仍有差距

#### Windows 已有

- 套索选择
- 对象复制
- 移动、缩放、旋转
- 组合、锁定
- 对齐和分布
- 层级调整
- 文本、图片和形状编辑

#### Android 当前状态

- 已支持自由形状混合套索，可选择笔迹、文字、图片和形状，并提供八类筛选、统一移动/缩放/旋转、复制、删除、组合/取消组合、层级、锁定、批量样式和跨页复制/移动。
- 文本、图片和形状的跨平台字段可以无损保存，图层显隐和透明度也会参与渲染。
- 选区现已可裁剪导出白底 PNG；主要差距收敛为键盘微调、更精细的控制柄命中、触控反馈、选区素材化和更复杂的组合编辑。

#### 建议

把对象编辑能力下沉为跨平台的 Core 命令和几何算法，由 Windows 与 Android 分别负责交互和渲染。至少统一：

- 点选和套索
- 选择框与控制柄
- 移动、缩放、旋转
- 复制、删除
- 颜色、线宽和透明度修改
- 前移、后移、置顶、置底
- 组合、取消组合、锁定
- 撤销/重做命令模型

基础闭环已经完成；后续重点应放在自由套索、笔迹级批量编辑和触控细节，而不是重复实现已有对象命令。

### 5.3 橡皮、选择和手写编辑：常用闭环已完成，精细交互仍有差距

Goodnotes、Notability 等产品的优势不只是“有套索”，而是选中后能快速完成筛选、复制、转换、缩放、改色、导出和跨页整理。

PaperNote 已完成：

- 自由形状混合套索；
- 全部内容、笔迹、钢笔、荧光笔、对象、文字、图片和形状八类筛选；
- 混合选区移动、缩放、旋转、复制、删除；
- 批量修改颜色、粗细、透明度和笔型；
- 跨页面复制/移动、组合/取消组合、层级和锁定；
- 混合选区裁剪导出白底 PNG，Android 通过系统分享保存或发送。

仍需补齐：

- 擦除时按钢笔/荧光笔或对象类型过滤；
- 选区保存为贴纸或常用元素，并支持透明背景、分辨率等导出选项；
- 键盘微调、更大的触控控制柄、震动/视觉反馈；
- 涂抹删除等自然手势；
- 误触保护、点击空白取消和复杂组合编辑细节。

### 5.4 页面、模板和资料库：基础与批量操作完整，规模化组织仍可深化

#### 已有

- 新建、复制、删除、重命名、移动和快速跳转页面；
- 多页复制、删除、移至首尾、书签、PDF 背景旋转、范围导出和范围提取；
- 缩略图、页码、页面搜索、手工大纲、原 PDF 目录和浏览历史；
- 空白、横线、方格、点阵、自定义图片纸张、收藏模板和跨笔记共享模板；
- 文件夹、标签、排序、最近笔记、回收站和多笔记标签页。

#### 仍缺

- 模板尺寸、方向、边距、背景色和线距的可视化编辑；
- 更完整的模板包导入/导出、分类和批量维护；
- Android 多页拖拽排序和跨笔记直接移动页面；
- 文件夹嵌套、收藏/置顶和智能筛选的进一步深化；
- 分屏或同时查看两个页面；
- 多个笔记合并与多来源 PDF 合并。

如果目标用户是一名学生，当前结构已经能覆盖“课程/文件夹 → 笔记本 → 页面 → 书签/大纲/标签”的日常主链路；下一步应优化大资料库效率，而不是继续增加零散层级。

### 5.5 PDF：学习资料主链路已完成，专业文档能力仍有差距

#### 已有

- Windows 与 Android 按页码范围导入，200 MB/500 页边界，异步逐页进度、取消、SHA-256 缓存和失败续接；
- 840 × 1188 JPEG 背景、原比例保持、书写/对象批注、旋转和裁剪；
- 带文本层 PDF 的离线文本提取、搜索结果片段和页码跳转；
- 原 PDF 目录/书签与内部 GoTo 链接，目标页已导入时可跳转；
- 统一批注列表、类型/颜色筛选和页面文字评论；
- 页面复制、删除、移至首尾、批量旋转、书签、范围 PDF 导出和提取为新 `.papernote`；
- 导出包含背景与批注的扁平 PDF，原 PDF 不被修改；
- PDF 文本、目录、内部链接和评论由格式 16 引入，当前格式 17 可继续跨 Windows/Android 保存。

#### 关键不足

1. **扫描资料识别**：没有文本层的扫描 PDF/图片仍缺离线 OCR。
2. **原文精细操作**：尚不能逐字选择/复制 PDF 原文，也没有直接基于原文字形的原生高亮；现有高亮是独立批注。
3. **更复杂页面处理**：尚未支持多个源 PDF 合并、页面替换、双页阅读和分屏；Android 批量操作使用页码范围而非复杂拖拽。
4. **批注管理深化**：目前可按类型/颜色筛选，但没有作者协作维度、审阅状态、批注回复线程或批注摘要导出。
5. **专业能力**：可编辑表单、数字签名、测量、比例校准和盖章尚未覆盖。
6. **极限规模验证**：200 MB/500 页边界仍缺更多低内存真机、低存储和长时间反复导入验证。
7. **导出选项**：仍可增加分辨率、压缩、是否包含背景/评论等细粒度设置。

结论：PaperNote 已经能完成“导入教材 → 按目录/搜索定位 → 书写和评论 → 按范围导出/提取”的离线学习闭环；下一阶段应优先补扫描 OCR 和原文选择，而不是追赶工程审图软件的全部功能。

### 5.6 搜索、OCR 和手写识别：文本检索已完成，识别引擎仍是最大效率差距

PaperNote 的结构化离线搜索已覆盖笔记/页面标题、标签、文本对象、PDF 文本、文字评论、来源名称和已保存的 OCR/手写识别结果，并可提供页码、来源和结果片段。

当前检索分层：

1. **第一级：文本索引（已完成）**
   - 索引笔记本标题、页面标题、文本对象、PDF 文本层和评论。
   - 支持结果片段、页码、来源和点击跳转。
2. **第二级：本地 OCR（未完成）**
   - 对扫描 PDF 和图片进行离线 OCR。
   - OCR 结果应保存为可重建索引，不破坏原始页面。
3. **第三级：手写搜索（未完成）**
   - 对中文、英文、数字和常见符号进行本地识别。
   - 保留识别置信度，避免把错误结果写回原笔迹。
4. **第四级：手写转文字与结构化编辑（未完成）**
   - 选中一段笔迹后转为文本。
   - 后续再扩展数学公式和图形识别。

#### 实施原则

- 不应从零训练完整识别模型；应评估可离线部署、许可可接受且能随安装包分发的 OCR/手写识别引擎。
- 索引必须可删除、可重建，并明确存放位置。
- 搜索不能依赖联网，也不能默认上传用户笔记。
- 识别失败不能影响原始笔迹和文件可打开性。

### 5.7 录音和课堂场景：不是基础刚需，但对学生价值很高

Notability 的代表优势是录音与笔迹时间轴：回放录音时可以定位当时写下的内容。Goodnotes 也已具备录音类工作流。

PaperNote 已完成本地课堂回放闭环：Windows 与 Android 都支持页面级录音、暂停/继续、播放、重命名、删除、命名时间标记和书写时自动记录时间点；本地波形可从开头、25%、50%、75% 跳转，播放经过时间标记时会高亮关联笔迹，音频附件也进入备份格式 3。

仍需补齐：

- 更完整的可拖动时间条、缩放波形和精确拖动定位
- 点击笔迹反向跳到对应录音时间
- 跨多个连续笔迹的更细粒度淡入/淡出回放
- 外接麦克风选择、实时音量提示和质量设置
- 存储空间预估、压缩质量选择和更强的异常中断恢复

不必做云端转写，也不必联网。

### 5.8 导入、导出、备份和数据安全：核心自救链路已完成

#### PaperNote 的优势

- 单笔记导入导出
- 整库备份和恢复
- 本地自动保存
- 回收站
- 历史版本
- 跨平台 PaperInk
- Android 无 INTERNET 权限

#### 已补强

- 临时文件写入、重新解析验证和原子替换
- 启动时自动恢复较新的可读草稿
- Windows/Android 恢复中心、损坏检测、只读诊断和另存抢救副本
- 备份格式 3 的清单、长度、SHA-256、重复路径和路径穿越校验

- ~~可选本地加密或密码保护界面~~（已完成：Windows/Android PNENC2）

一个本地优先产品最重要的承诺不是“数据在本地”，而是“数据在本地并且不容易丢”。

### 5.9 Windows 与 Android 一致性：共享格式是优势，交互能力尚未统一

PaperNote 已经采用共享 Core 和 PaperInk，这是后续扩展的正确基础。但目前存在“格式一致、能力不一致”的问题。

需要建立跨平台能力合同：

- 同一对象在两个平台都能创建、查看、编辑和删除
- 同一笔迹在两个平台渲染尺寸、颜色和压感结果一致
- 同一 PDF 页裁剪、旋转和缩放结果一致
- 撤销/重做语义一致
- 旧版本文件升级后两端结果一致
- 不支持的对象也必须无损保留，不能打开后保存就丢失

建议为每个文件格式版本建立 Windows ↔ Android 往返测试：创建、保存、互相打开、编辑、再次保存，再比较文档模型和渲染快照。

## 6. PaperNote 当前真正的优势

与竞品比较时，不应只列缺点。PaperNote 已经有几项值得继续强化的产品定位：

1. **本地优先且无强制账号**：打开即可用，用户自己决定文件位置和备份方式。
2. **开源与可审计**：适合关心隐私、希望长期维护或自行定制的用户。
3. **Windows + Android 共用数据模型**：跨平台不是重新做两个互不兼容的应用。
4. **PaperInk 跨平台墨迹**：已经为后续平台扩展和数据迁移打下基础。
5. **PDF、页面对象和资料库已经形成雏形**：不是单一白板或单页画板。
6. **回收站、历史版本、整库备份已具备基础**：比许多小型开源笔记项目更接近日常使用。
7. **Android 不申请 INTERNET 权限**：这是可以被用户直接验证的隐私承诺。
8. **没有订阅、广告和数据商业化压力**：可以把工程资源放在可靠性和离线体验上。

建议公开定位为：

> 面向 Windows 与 Android 的开源、本地优先手写笔记与 PDF 批注工具。

不要宣传为“完整替代 Goodnotes”，至少在 Android 对象编辑、全文检索、大文档可靠性和 PDF 导航完成之前不宜这样承诺。

## 9. 建议的产品验收场景

后续开发不要只按按钮数量验收，应按完整任务验收。

### 场景 A：课堂讲义

1. 导入 200 页 PDF。
2. 通过目录和搜索定位章节。
3. 连续书写 90 分钟。
4. 使用钢笔、荧光笔、橡皮和套索整理。
5. 添加页面和图片。
6. 关闭应用并重新打开，回到原页和原缩放位置。
7. 导出指定页范围，批注位置保持准确。

### 场景 B：Windows 与 Android 往返

1. Windows 创建文本、图片、形状和手写内容。
2. Android 打开并移动、缩放、改色和继续书写。
3. Android 保存后在 Windows 打开。
4. 检查对象、字体、颜色、压力、裁剪、层级和页序一致。

### 场景 C：故障恢复

1. 保存过程中强制结束应用。
2. 模拟磁盘空间不足。
3. 导入损坏 PDF。
4. 打开旧数据格式文件。
5. 从历史版本恢复。
6. 校验最后一个有效版本仍可打开，失败操作不会污染原文件。

### 场景 D：长期资料库

1. 创建 50 个课程笔记本、1000 个页面。
2. 使用文件夹、标签、书签和大纲组织。
3. 搜索文本、PDF 和 OCR 内容。
4. 备份整库并在空目录恢复。
5. 比较恢复前后的文件数、页数、对象数和校验值。

## 10. 最终判断

PaperNote 现在最接近的是：

> 一个已经具备真实本地资料库、跨平台手写、PDF 文本与导航、批注管理、页面批量处理和可靠自救链路的 Windows/Android 开源笔记应用。

它仍不是 Goodnotes 的完整离线替代品，但差距已经从“缺少 PDF 学习主链路”转为更难的精细能力：

- 扫描 PDF/图片离线 OCR、手写搜索/转文字和数学识别；
- 标尺、几何吸附、自动形状识别、选区导出和更细腻的书写质感；
- 音频波形与笔迹同步回放；
- 双页/分屏、完整无障碍、高对比度和键盘效率；
- PDF 原文逐字选择、表单、签名、测量、多源合并和更丰富导出选项；
- 更多低内存 Android 真机和极限规模长期验证。

最合理的后续顺序是：

1. 接入可随安装包离线分发的 OCR，引导扫描资料进入现有索引；
2. 完成选区导出、标尺/吸附和更精细的书写工具；
3. 把已有录音时间标记做成可见波形与同步回放；
4. 系统完成键盘、屏幕阅读器、高对比度和触控目标验收；
5. 最后再扩展双页/分屏、插件与高级 PDF 专业工具；笔记级加密已提前完成。

当前项目的最大优势仍不是复制所有商业功能，而是以开放文件、共享 Core、离线处理和可验证备份提供长期数据控制；新增功能必须继续满足“可保存、可恢复、可测试、跨平台不丢数据”。

## Implementation status — 2026-07-23

本节是当前离线版本能力边界的统一说明。

### 已实现并纳入仓库测试

- Windows 与 Android 共用 PaperInk 和页面对象模型；支持压力、倾角、平滑、整笔/局部擦除、透明度、图层及自由形状混合套索。
- 页面支持复制、删除、移动到开头/末尾、书签、PDF 背景旋转，以及按页码范围导出 PDF 或提取为新的 `.papernote`；提取时会生成新 ID 并重映射保留下来的内部链接。
- Windows 与 Android 的 PDF 导入支持 200 MB/500 页边界、逐页进度、取消、完整 SHA-256 内容指纹缓存、失败续接和旧缓存清理；原 PDF 始终不被修改。
- 对带文本层的 PDF，会在本机离线提取并保存页面文本、原 PDF 目录/书签和内部 GoTo 链接；Windows 与 Android 均可搜索文本、浏览目录并跳转到已导入的目标页。
- PDF 学习工作流包含统一批注列表、类型/颜色筛选和页面文字评论；评论、PDF 文本、目录和链接都保存在 `.papernote` 中，可随 Windows/Android 文件往返。
- 离线搜索覆盖笔记/页面标题、标签、文本对象、PDF 文本、文字评论、来源名称以及已保存的 OCR/手写识别结果。
- 保存采用“临时文件写入并验证后替换”；Windows 与 Android 均提供恢复中心、只读损坏检查和另存抢救副本。
- 整库备份格式 3 包含笔记、历史版本和音频附件，并校验长度与 SHA-256；页面级本地录音支持播放、暂停、重命名、删除、时间标记、本地波形、分段跳转、播放笔迹高亮和选中笔迹反向跳转。
- Windows 与 Android 均提供本机个人素材库，可把混合选区保存为最多 48 项可复用素材；重复插入会生成新内容 ID、重映射对象组并自动缩放。
- 当前 `.papernote` 数据格式版本为 17；加载器会修复常见无效 ID、坐标、图层引用、链接、目录、评论和波形数据。
- 后台回归覆盖 Core、真实 PDF 内容提取、格式 17 存储往返、几何规整、混合选区 PNG、录音波形、播放笔迹高亮、Windows 隐藏 UI、Android Release/AOT 与 APK 静态检查；测试过程不操控用户鼠标。

### 当前明确边界

- PDF 文本搜索依赖原文件自带文本层；扫描版 PDF 和图片尚未内置离线 OCR，因此只能作为页面图像批注。
- PDF 内部链接只有目标页也已导入到当前笔记时才能跳转；未导入目标会明确提示。
