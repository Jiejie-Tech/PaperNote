# PaperNote 离线本地竞品差距分析

> 评估日期：2026-07-23
> 评估对象：PaperNote 1.0.0（Windows、Android）
> 范围：只比较无需账号、云同步、多人协作、在线素材商店或联网 AI 的本地能力。

## 1. 结论

PaperNote 已完成此前列出的 P0、P1、P2 离线主链路，不再只是“能写字和导入 PDF”的原型。它现在覆盖书写、对象/图层、页面组织、可恢复 PDF 导入、离线 OCR、基础公式、表单/签名覆盖层、测量与几何、笔记合并、双页/参考阅读、录音时间轴、素材/模板资源包、安全声明式扩展、恢复/备份和跨平台数据往返。

相对 Goodnotes、Notability、OneNote、Xodo 等成熟产品，PaperNote 的主要差距已经从“功能缺失”转为三类：

1. **精细体验**：笔刷质感、低延迟调校、套索高级操作、同屏多文档和移动端长时间稳定性。
2. **专业深度**：成熟数学识别、PDF 原生文本选择/高亮、AcroForm 写回、证书数字签名和精细音频编辑。
3. **验证规模**：多品牌触控笔、低内存 Android、超长录音、数百页复杂 PDF 和完整无障碍认证。

PaperNote 的稳定差异化是：**完全离线、无账号、无广告、无遥测、Android 无网络权限、开放格式实现、Windows/Android 共用数据模型。**

## 2. 评估口径

纳入：手写、笔刷、橡皮、套索、对象、图层、页面、模板、PDF、OCR、搜索、公式、录音、备份、恢复、加密、性能和无障碍。

排除：账号、厂商云同步、多人实时协作、网页版、在线 AI、在线商店、订阅内容和社交社区。排除项不是 PaperNote 的缺点。

## 4. 与 Goodnotes 的差距

### 已基本追平的离线主链路

- 笔记本/页面/模板组织、PDF 导入批注、搜索、书签/目录和范围导出。
- 钢笔/荧光笔/橡皮、压力与倾角、混合套索、文本/图片/形状、图层和对象组合。
- 离线 OCR、手写转文字、录音与笔迹时间关联、个人素材复用。
- 双页/参考阅读、页面合并、PDF 覆盖层表单/签名/测量和本地密码保护。

“基本追平”指 PaperNote 已有可用闭环，不代表交互细腻度、识别准确率或大规模稳定性达到商业产品水平。

### 仍明显不足

1. **书写手感**：缺少成熟产品多年积累的渲染、预测、笔刷纹理、速度曲线和硬件适配。
2. **数学识别**：现在只是基础规则转换，不能等同于成熟的手写公式识别。
3. **PDF 原生编辑**：覆盖层不会写回 AcroForm，也没有证书数字签名和逐字原生高亮。
4. **同屏多任务**：现有双页/参考视图以只读为主，Android 还不是可编辑分屏。
5. **录音编辑**：Windows 没有可靠设备枚举和质量档，裁剪是播放范围而非重写音频。
6. **无障碍与真机矩阵**：有基础实现和自动化检查，但缺少系统化认证与多设备长期测试。

## 5. 与 Notability 的差距

PaperNote 已有录音波形、时间标记、播放笔迹高亮和笔迹反向跳转；Android 还有三档质量。差距集中在波形缩放、选区、静音检测、音频段落编辑、Windows 输入设备选择和课堂级长录音稳定性。PaperNote 的优势是录音及关联元数据完全本地，并可随备份包迁移。

## 6. 与 OneNote 的差距

PaperNote 在自由画布、跨端文件往返、离线搜索和页面对象方面已有基础；但没有 OneNote 式无限画布、复杂文本排版、表格、Office 嵌入和成熟键盘工作流。因为本项目聚焦分页纸张与 PDF，这些差距不全部需要补齐。

## 7. 与 Xodo 等 PDF 工具的差距

PaperNote 更适合“讲义变成分页笔记”的学习流程，但不是专业 PDF 编辑器。它不直接修改原 PDF、AcroForm、证书签名或原生文字对象；优势是批注、录音、手写、OCR、页面对象和笔记结构统一保存在可迁移的 `.papernote` 中。

## 8. 当前剩余优先级

### P0：发布质量与真机稳定性

- 低内存手机、平板、主流触控笔、多品牌 Android 真机矩阵。
- 200 MB/500 页复杂 PDF、超长录音、前后台切换、空间不足和异常退出压力测试。
- 崩溃日志的本地导出和更系统的故障注入。

### P1：高频体验

- 笔刷质感和速度响应、套索对齐分布/筛选、Android 同屏参考编辑。
- Windows 录音设备与质量设置、波形缩放和更精细时间轴。
- PDF 逐字选择、复制选区和原生高亮。

### P2：专业增强

- 更成熟的离线数学/手写模型。
- AcroForm 写回、证书型数字签名、复杂表单外观。
- 完整 TalkBack/Narrator/WCAG 专项验收。

## 9. 验收结论

此前竞品报告中要求完成的 P0、P1、P2 **本地功能项已在代码和后台测试中闭环**。接下来最重要的不是继续堆菜单，而是用真机、真实长笔记和大 PDF 做长期稳定性验证，并逐步提升书写、选择、录音和无障碍的精细体验。

## Implementation status — 2026-07-23

本节是当前离线版本能力边界的统一说明。

### 已实现并纳入仓库测试

- Windows 与 Android 共用 PaperInk、页面对象和格式 19 数据模型；支持压力、倾角、平滑、整笔/局部擦除、透明度、图层、对象组合、锁定、层级和自由形状混合套索。
- 几何工具覆盖直线与 45° 吸附、矩形/椭圆规整、长度测量、可设长度和角度的标尺线、角度辅助线、平行线及垂线构造。
- PDF 支持 200 MB/500 页边界、范围导入、进度/取消/失败续接、SHA-256 缓存、原文与字词坐标提取、搜索/复制、目录、内部链接、评论、批注筛选、页面旋转、范围导出与提取。
- 高级 PDF 覆盖层支持文本填写框、复选框、必填校验、图片/选区签名和测量比例校准；可合并本地笔记本并安全重映射页面、图层、笔迹、对象、链接、目录和录音时间点 ID。
- 本地识别支持当前页与整本批量 OCR、进度/取消、区域/置信度/校对状态保存、选区手写转文字；基础数学规则可生成并编辑 LaTeX，复制公式并导出 SVG。
- Windows 提供双页只读阅读窗口和独立参考页窗口；Android 提供双页/参考页只读视图，返回后继续编辑原页。两端均可保存工具栏布局，移动端关键按钮保持至少 48 dp 触控尺寸。
- 个人素材库和共享模板支持分类、搜索、收藏及本地资源包导入/导出；声明式本地扩展只允许受限文字操作，不加载 DLL、不运行脚本或第三方代码。
- 页面录音支持波形、播放/暂停、时间标记、笔迹双向跳转、标记移动/重命名/删除和非破坏性播放裁剪；Android 另提供省空间、标准、高质量三档录音质量。
- 保存采用临时文件验证后替换；恢复中心、只读损坏检查、历史版本、整库备份格式 3、PNENC2 笔记级密码保护和 PNENC1 兼容均已纳入测试。
- 当前 `.papernote` 内部格式为 `FormatVersion 19`：格式 17 引入录音波形，格式 18 引入批量 OCR 元数据，格式 19 引入 PDF 字词坐标/测量比例、表单/签名/公式元数据和录音质量/裁剪字段。
- 后台回归覆盖 Core、格式 19 往返、PDF/公式/合并/资源包/安全扩展、Windows 隐藏 UI、Android Release 编译与 APK 静态检查；整个测试过程不操控用户鼠标。

### 当前明确边界

- 数学识别是基础规则转换，不是成熟公式识别模型；复杂手写公式仍需手动编辑 LaTeX。
- PDF 表单、签名和测量以 PaperNote 页面覆盖层保存，不直接写回原 PDF 的 AcroForm；本地签名是图片签名，不是证书型数字签名。
- PDF 内部链接仅在目标页已导入当前笔记时可跳转；原文可查看和复制，但尚未提供逐字选择、原生文本高亮或直接修改原 PDF。
- Windows 录音使用系统默认录音设备，目前没有可靠的设备枚举和三档编码质量；Android 已提供三档质量。播放裁剪不会改写原音频文件。
- 双页与参考视图是只读辅助阅读：Windows 主窗口可继续编辑，Android 返回编辑器后继续编辑原页，不是同屏双画布编辑。
- 扩展系统只执行声明式文字命令；不会加载不受信任代码。无障碍提供基础语义、键盘焦点和高对比度资源，但未宣称通过完整 WCAG、TalkBack 或 Narrator 专项认证。
- 200 MB/500 页边界已自动化验证；低内存 Android、多品牌触控笔、超长录音和长时间运行仍需要持续真机压力测试。
- 账号、云同步、联网 AI、遥测、广告、多人协作和在线商店不在本地离线版本范围内。
- APK、ZIP、签名密钥、构建输出和私人笔记属于发布产物或本地数据，不提交到源码仓库。
